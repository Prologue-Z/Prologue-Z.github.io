# 📋 个人主页项目 - 工作流程

## 一、内容更新流程

### 1.1 添加博客文章
```bash
# 1. 创建新文章
cd ~/.openclaw/workspace/projects/个人主页
hexo new post "文章标题"

# 2. 编辑文章
# 编辑 source/_posts/文章标题.md

# 3. 预览
hexo server

# 4. 部署
ssh-agent bash -c 'ssh-add ~/.ssh/hexo-deploy-new && hexo clean && hexo generate && hexo deploy'
```

### 1.2 修改页面内容
```bash
# 1. 编辑页面文件
# 例如：source/about.md

# 2. 预览
hexo server

# 3. 部署
ssh-agent bash -c 'ssh-add ~/.ssh/hexo-deploy-new && hexo clean && hexo generate && hexo deploy'
```

## 三、GitHub 配置流程

### 3.1 添加 Deploy Key
1. 打开 https://github.com/Prologue-Z/Prologue-Z.github.io/settings/keys/new
2. Title: `hexo-deploy-key`
3. Key type: **Write access** ✅
4. 粘贴公钥：`~/.ssh/hexo-deploy-new.pub`
5. 点击 **Add deploy key**

### 3.2 配置 GitHub Pages
1. 打开 https://github.com/Prologue-Z/Prologue-Z.github.io/settings/pages
2. Source: **Deploy from a branch**
3. Branch: **gh-pages**
4. Folder: **/ (root)**
5. 点击 **Save**

## 五、故障排查流程

### 5.1 部署失败
```bash
# 1. 检查 SSH key
ssh -T git@github.com

# 2. 重新添加 key
ssh-add ~/.ssh/hexo-deploy-new

# 3. 重新部署
ssh-agent bash -c 'ssh-add ~/.ssh/hexo-deploy-new && hexo clean && hexo generate && hexo deploy'
```

### 5.2 页面不更新
1. 检查是否推送到 gh-pages 分支
2. 等待 5 分钟让 GitHub Pages 刷新
3. 清除浏览器缓存
4. 使用无痕模式访问

### 5.3 主题不生效
1. 检查 `source/_config.cactus.yml` 配置
2. 运行 `hexo clean` 清除缓存
3. 重新生成并部署
